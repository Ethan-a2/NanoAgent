## 1.1 项目概述

### 1.1.1 项目定位

**NanoAgent** 是一个 **135M 参数** 的轻量级语言模型（SLM），专为 **agentic 任务** 设计：
- 工具调用（Tool Calling）
- 指令跟随（Instruction Following）
- 轻量级推理（Lightweight Reasoning）

### 1.1.2 核心特性

| 特性 | 说明 |
|------|------|
| 参数量 | 135M |
| 上下文长度 | 8k |
| 部署平台 | 边缘设备（笔记本、CPU、可穿戴设备） |
| 训练方法 | SFT + GRPO |
| 训练框架 | PyTorch (CUDA) |

### 1.2.2 为什么使用 PyTorch + CUDA？

- **GPU 加速**：充分利用 NVIDIA GPU 并行计算能力
- **生态丰富**：支持 HuggingFace、DeepSpeed、FSDP 等
- **易于部署**：支持量化、TensorRT、Triton 等优化

### 1.2.3 训练策略

```
SFT (Supervised Fine-Tuning) → GRPO (Group Relative Policy Optimization)
```

1. **SFT 阶段**：基础能力学习（对话、推理），使用 PyTorch + CUDA
2. **GRPO 阶段**：强化工具调用能力

## 1.4 关键技术决策

### 1.4.1 工具调用格式

```json
[{"name": "tool_name", "arguments": {"arg1": "val1"}}]
```

### 1.4.2 训练技巧

| 技术 | 作用 |
|------|------|
| Gradient Checkpointing | 减少显存占用 |
| Dynamic Fine-Tuning (DFT) | 稳定训练 |
| KL Divergence | 防止灾难性遗忘 |
| Cosine Learning Rate | 平滑收敛 |

### 1.4.3 推理采样

- Temperature: 0.3-0.4
- Top-p: 0.9
- Top-k: 50
- Min-p: 可选

## 1.6 总结

NanoAgent 是一个针对 **边缘设备** 和 **工具调用** 场景优化的 135M 参数模型：

1. **架构简洁**：单一模型，无多模态

3. **专注任务**：工具调用 + 指令跟随
