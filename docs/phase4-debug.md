## 4.1 性能瓶颈

### 4.1.1 训练瓶颈

| 瓶颈 | 原因 | 解决方案 |
|------|------|----------|
| 显存不足 | 135M 模型 + 激活值 | 梯度检查点、量化 |
| 训练速度慢 | 单批次训练 | 增大 batch_size（需要更多显存） |
| 收敛慢 | 学习率设置不当 | 使用 cosine scheduler |
| 梯度爆炸 | 初始化问题 | gradient clipping (max_norm=1.0) |

### 4.1.2 推理瓶颈

| 瓶颈 | 原因 | 解决方案 |
|------|------|----------|
| 生成慢 | 连续 token 生成 | KV cache 优化 |
| 内存占用高 | 全量模型加载 | 量化（4-bit/8-bit） |
| JSON 解析失败 | 输出格式错误 | prompt engineering |

### 4.1.3 数据瓶颈

| 瓶颈 | 原因 | 解决方案 |
|------|------|----------|
| 数据加载慢 | 磁盘 IO | 使用 pickle 缓存 |
| 数据不平衡 | 数据集分布不均 | 重采样/shuffle |

## 4.3 常见问题

### 4.3.1 模型不收敛

**症状**：训练 loss 震荡或不变

**排查步骤**：
1. 检查学习率是否设置正确（建议 1e-4 ~ 1e-5）
2. 检查数据是否正确加载（token 数量）
3. 检查梯度是否正常（是否有 NaN）
4. 检查权重是否正确更新

### 4.3.2 工具调用失败

**症状**：模型不生成 JSON 格式的工具调用

**排查步骤**：
1. 检查 prompt 模板是否正确
2. 检查工具描述是否完整
3. 检查 temperature 是否过高（建议 0.3-0.4）
4. 使用 `do_sample=False` 调试

### 4.3.3 显存 OOM

**症状**：`MemoryError` 或进程被 kill

**排查步骤**：
1. 启用梯度检查点（`GRADIENT_CHECKPOINT_LAYERS`）
2. 减小 `CONTEXT_LEN`
3. 使用模型量化（`QUANTIZATION=4`）
4. 减小 `BATCH_SIZE`

## 4.5 性能优化技巧

### 4.5.1 训练优化

| 技巧 | 效果 | 注意事项 |
|------|------|----------|
| 梯度检查点 | 减少 50% 显存 | 轻微增加训练时间 |
| 量化 | 减少 75% 显存 | 可能影响精度 |
| 混合精度 | 减少 50% 显存 | 需要硬件支持 |
| 动态填充 | 减少 10-20% tokens | 实现复杂 |

### 4.5.2 推理优化

| 技巧 | 效果 | 注意事项 |
|------|------|----------|
| KV Cache | 加速生成 | 内存占用增加 |
| 量化 | 减少内存 | 可能影响质量 |
| Batch 推理 | 吞吐量增加 | 延迟增加 |
| 量化注意力 | 减少计算 | 实现复杂 |
